Title & thumbnail A/B testing · Helm Signal (v1.5)

Let the numbers pick the packaging.

ChannelHelm already generates several scored title options and multiple AI thumbnail concepts per video. A/B testing closes the loop: it rotates those variants on your live YouTube video, reads each one's real performance from the YouTube Analytics API, applies the winner permanently, and feeds the result back so the next video's titles lean toward what actually works.

Why self-run

YouTube's native test isn't in the API.

YouTube Studio has a built-in “Test & compare” for thumbnails (and, increasingly, titles) that picks a winner by watch-time share. It is Studio-UI only — the YouTube Data API exposes no way to create or read those tests. The API gives you exactly two levers: thumbnails.set (one thumbnail) and videos.update (the title).

So ChannelHelm runs the test itself: it rotates the variants on a schedule and decides the winner from the YouTube Analytics API. You trade YouTube's watch-time-based winner selection for full automation and a result that flows straight back into generation.

Trade-off Self-run rotation is noisier than YouTube's native test (each variant sees a different slice of time rather than a randomized concurrent split), and YouTube Analytics has a 1–3 day data latency. Use generous rotation windows (the 48-hour default) and the min_views guardrail so a decision rests on enough data.
How it works

One worker, one state machine.

A recurring experiment_tick job advances each running experiment. It only acts when the current variant's window has elapsed, so ticking often is cheap.

apply variant A→ wait rotation_hours→ read Analytics→ rotate B…N→ repeat × rounds→ apply winner ✓
  1. Start. On launch, variant A's title and/or thumbnail goes live on the video.
  2. Run the window. The variant stays up for rotation_hours (default 48).
  3. Observe. When the window closes, ChannelHelm reads that window's views, watch time, and — when your channel reports them — impressions and impressionClickThroughRate from the Analytics API, and stores them on the variant.
  4. Rotate. The next variant goes live; repeat until every variant has run, for as many rounds as you set.
  5. Decide. Once every variant has completed its rounds and cleared min_views, the highest-scoring variant on your chosen metric wins (ties broken by total views).
  6. Apply & learn. The winner is set permanently, and the result feeds voice_examples — see the feedback loop.

The deciding metric

MetricWhat it measuresWhen to use it
views (default)Views per day in each window (normalized so uneven windows compare fairly).Always available. The robust default.
impression_ctrImpression-weighted click-through rate. Falls back to views/day if your channel doesn't report impressions.The truest packaging signal — when available.
estimated_minutes_watchedTotal watch time across the variant's windows.When you care about retention, not just the click.
Setup

Grant the Analytics scope.

Reading performance needs the YouTube Analytics scope, which is broader than the upload scope used for publishing. ChannelHelm requests yt-analytics.readonly at connect time. 

# scopes ChannelHelm requests per brand
youtube.upload         # publish the video
youtube                # set thumbnail + update title (A/B rotation)
yt-analytics.readonly  # read impressions / CTR / views (A/B decision)
Reconnect Brands connected before v1.5 don't have the analytics scope. The A/B panel shows an amber banner when a brand needs to reconnect — open the brand page, click Disconnect, then Connect YouTube again and accept the new permission. The connection status reports analytics: true once it's granted.
Run a test

From the package Studio.

A/B tests run against a video that's already published via YouTube Direct — that's how ChannelHelm gets the video id to rotate. If the package isn't live yet, the panel explains what to do first.

  1. Open the package and find the A/B Tests panel in the Studio.
  2. New A/B test. Pick what to test — Title, Thumbnail, or both.
  3. Assemble 2–3 variants. For each, choose one of the generated scored title options and/or one of the AI thumbnail concepts. Every variant must change something.
  4. Set the run. Deciding metric, rotation hours (default 48), rounds (default 1), and min_views (default 50).
  5. Launch. Variant A goes live immediately; the rest follow on the schedule. The panel shows each variant's accumulated numbers, the current live variant, and the winner once decided.
Cost A/B testing adds no LLM or image spend — it reuses titles and thumbnails you already generated. The only cost is YouTube API quota (cheap: a few units per rotation + per analytics read).
The feedback loop

Winners teach the next video.

When a title experiment decides, the winning title is written to voice_examples with a high performance score, and the losing titles with a low one. Because generate_asset pulls top-scored voice examples as few-shot exemplars, the next package's titles drift toward the packaging your audience actually rewards — the core of the Helm Signal loop.

A/B winner→ voice_examples (+/−)→ generate_asset few-shot→ better next title
Compounding Every decided test makes the title generator a little sharper. Thumbnail-style learning (feeding the winning concept back into thumbnail prompts) is the next slice on the roadmap.
